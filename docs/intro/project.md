# 项目介绍

DoneMail 是基于 Cloudflare Email Routing 和 Workers 构建的高性能私有邮箱服务。

## 为什么是 DoneMail

DoneMail 适合把邮箱当成自动化入口，而不是只当作人工阅读工具。

你可以把多个域名接入 DoneMail，让所有地址都进入同一个控制台；用检索快速定位历史邮件；用策略把重要邮件推送到 Telegram、HTTP Webhook 或其它邮箱；用共享邮件或共享账户把指定内容交给外部人员查看；用公开 API 把邮件能力接入脚本、监控或业务系统。

## 功能概览

| 功能 | 说明 |
| --- | --- |
| 配置中心 | 连接 Cloudflare、设置入口域名、邮件定时清理（保留天数最少 1 天） |
| 全域收信 | 接入域名后，可接收该域名下所有地址的邮件，不需要逐个创建邮箱账号 |
| 高性能检索 | 主题、地址与正文 FTS，中文补充分词，游标分页列表；收信后即可检索 |
| 自动化策略 | 按发件人、收件人、主题、正文、附件等条件触发转发 / HTTP / Telegram |
| 安全共享 | 为单封邮件或邮箱账户生成带有效期的共享链接，默认不加载远程图片 |
| 公开接口 | 使用管理员 Key 对接收信、搜索、附件和共享能力 |
| 策略转发 | 收信时按规则转发到已验证邮箱（非主动发信） |
| 异常记录 | 仅记录域名/策略/公开接口/系统清理等失败与拦截，不记成功流水 |
| 检查更新 | 对照上游 GitHub Release，便于确认是否需要同步自己的 Fork |

## 项目边界

DoneMail 保持单管理员、自托管和 Cloudflare-first。

它不追求企业套件式的大而全，不内置多用户注册系统，不做复杂权限体系，也不把自己包装成传统邮箱服务器。**不提供主动写邮件 / 发件箱**（策略「转发」除外）。项目优先保证配置集中、检索高效和自动化能力清晰可控。
